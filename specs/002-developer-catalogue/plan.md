## Summary

Build the developer catalogue on `/dashboard`: React Query fetches 22+ developer profiles from JSON Server (`GET /developers`), caches them for 5 minutes, and renders a responsive card grid. A search field and multi-select seniority toggle (Junior / Mid / Senior) filter the cached list client-side via `useMemo`. Loading state uses skeleton cards; errors show a retry button. All filtering is synchronous on cached data — no server-side filtering, no debouncing.

## Implementation Notes

### QueryClient Provider
`app/providers.tsx` is a `'use client'` component wrapping children with `<QueryClientProvider>`. It is imported in `app/(private)/layout.tsx` so the QueryClient is scoped to authenticated routes. This avoids instantiating React Query on the public login page.

### Filter Logic
`CatalogueView.tsx` owns `filterState` (via `useState`) and derives `filteredDevelopers` via `useMemo`. This component is `'use client'`; `page.tsx` remains a Server Component that simply renders `<CatalogueView />`.

### Seniority Toggle State
Seniority toggle: `seniorities: Seniority[]` — empty = all shown. Toggling an active seniority removes it from the array; toggling inactive adds it. No "all" button needed — deselecting all naturally restores the full list.

### Avatar Fallback
Each `<DeveloperCard>` uses `<img onError={e => { e.currentTarget.src = \`https://api.dicebear.com/7.x/initials/svg?seed=\${name}\` }}>`. DiceBear initials SVG generated from the developer's name provides a consistent, attractive fallback.

### Skeleton Cards
`CatalogueGrid.tsx` renders 8 skeleton `<div>` placeholders with `animate-pulse` while `isLoading` is true. Fixed dimensions match the real card to prevent layout shift (SC-004).
