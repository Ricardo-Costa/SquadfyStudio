## Summary

Implement the squad builder on `/dashboard`: a `SquadProvider` (Context API + useReducer) holds
global squad state (`members: Developer[]`, max 5). The dashboard splits into a two-column layout
(catalogue ~70% left, squad panel ~30% right sidebar on desktop; stacked on mobile). `DeveloperCard`
receives `isInSquad`, `isFull`, and `onAdd` props ‚Äî showing a badge or disabled/active add button.
`SquadPanel` displays squad members with a remove action per entry and a capacity indicator ("X/5").
All state logic lives in the reducer; the hook exposes a clean interface consumed by the two
client components.

## Implementation Notes

### SquadProvider in layout
`app/(private)/layout.tsx` wraps children with both `<Providers>` (React Query) and `<SquadProvider>` ‚Äî in that order so React Query is outermost, squad innermost. Both are needed on all private routes.

### CatalogueView squad integration
`CatalogueView` already calls `useDevelopers()`. It now also calls `useSquad()` to get `isFull` and `isMember`. For each developer in `filteredDevelopers`, it passes `isInSquad={isMember(dev.id)}`, `isFull={isFull}`, and `onAdd={addMember}` to `DeveloperCard`. No new `useMemo` needed ‚Äî these are O(5) lookups.

### DeveloperCard button logic
Three exclusive states based on `isInSquad` + `isFull`:
1. `isInSquad` ‚Üí emerald "No Squad" badge (no button)
2. `!isInSquad && isFull` ‚Üí disabled button with reduced opacity + `cursor-not-allowed`
3. `!isInSquad && !isFull` ‚Üí active button calling `onAdd(developer)` on click

### SquadPanel layout
Fixed sidebar at `340px` on desktop (`lg:grid-cols-[1fr_340px]`). Sticky positioning so it stays visible while scrolling the catalogue. On mobile, full-width below catalogue.

### SquadMemberCard
Simple component: avatar, name, seniority chip, `$cost/hr`, and a remove `<button>` (√ó) calling `removeMember(id)`.
